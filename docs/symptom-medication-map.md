# 症状→药品/建议 对照表（依《伤寒论》条文）

> 用于"缺药提示"功能。修改本文件即可调整对应关系，无需改代码。
> 代码读取本文件底部 JSON 数据段。
> 判定核心：先分**常备药范畴**与**其他**，常备药范畴内再按具体症状提示用药类型；其他一律提示"需辨证就医"。

---

## 一、判定总则

《伤寒论》中许多症状是主证"附带表现"，治主即解，不单独用药（如恶寒随表解、身痛随汗解、口渴随热清）。
另一些症状有明确对症成药，属感冒期间家庭常备药范畴。
其余不能以常备成药解决者，病机需辨证（寒热虚实）才能定方，统一提示"需辨证就医"。

故分四类：
1. **drugCovers**：现有硬编码药品的生效覆盖范围（用于判断已服药是否覆盖某症）。
2. **needsDrug（常备药范畴）**：可按具体症状提示用药类型（模糊，不点名具体药）。
3. **needsConsult（非常备药）**：统一提示"需辨证就医"。
4. **selfResolving（自愈随主解）**：治主即解，不单独用药，仅给护理建议。
5. **seeDoctor（危险症状）**：优先提示就医。

## 四、非常备药（needsConsult）— 统一提示"需辨证就医"

以下症状病机需辨证（寒热虚实相反，常备成药无法应对），一律提示需辨证就医，不笼统推荐某药：

- 腹满
- 腹痛
- 心下痞
- 心下悸
- 胸胁苦满
- 便秘（有腑实象）
- 不欲饮食

---

## 六、危险症状（seeDoctor）— 优先提示就医

`谵语`、`身黄`、`手足逆冷`、`蜷卧`、`喜唾`

---

## 七、显示口径

按优先级判断每条症状归属：
1. **已服药生效范围覆盖**该症状 → 不提示。
2. 属 **seeDoctor**（危险症状） → 红字提示尽快就医。
3. 属 **needsDrug**（常备药） → 模糊提示"需要 XX 药"，不点名具体药品。
4. 属 **needsConsult**（非常备药） → 提示"需辨证就医"。
5. 属 **selfResolving**（自愈） → 显示护理建议，不提示备药。

## 八、JSON 数据源（权威）

实际被代码读取的数据在 `src/data/symptomMedication.json`。
本文档第五/三/四节的表格为人工可读说明，**修改对应关系请改 JSON 文件**，本文档仅作参考。

JSON 字段：
- `drugCovers`：每个药品 id 覆盖的症状数组（用于判定已服药生效范围是否覆盖某症）。
- `antipyreticDrugIds`：含退烧成分的药品 id（判定缺退烧药）。
- `needsDrug`：常备药范畴，症状 → 用药类型（模糊，不点名具体药）。
- `needsConsult`：非常备药症状数组，统一提示"需辨证就医"。
- `selfResolving`：自愈随主解，症状 → 护理建议。
- `seeDoctor`：危险症状数组，优先提示就医。
