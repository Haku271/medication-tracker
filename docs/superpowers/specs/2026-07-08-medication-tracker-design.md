# 退烧药与感冒病程追踪器 - 设计文档

## 项目概述

一款纯前端的移动端 Web 应用，用于记录感冒病程、追踪退烧药服用情况，并基于《伤寒论》规则自动推断感冒所处阶段。

---

## 核心功能

### 1. 首页（感冒详情页）
- **有进行中的感冒**：显示当前感冒详情（症状时间轴、阶段推断、服药状态、体温记录、备注）
- **无感冒**：显示历史感冒列表 + 创建按钮 + 导出功能

### 2. 创建感冒
- 时间选择：当前时间 / 自定义时间
- 初始症状：多选，可跳过
- 确认后进入详情页

### 3. 症状记录
- 多选症状：恶寒、发热、头痛、鼻塞、流涕、咽痛、咳嗽、乏力、肌肉酸痛、往来寒热
- 可选体温记录
- 时间：当前 / 自定义
- 备注

### 4. 服药管理
- 内置 5 种退烧药，预置参数
- 服药后实时计算：生效状态、距离下次可服时间

### 5. 伤寒论阶段推断
- 根据症状时间轴，按配置规则自动推断当前阶段
- 基础版阶段：太阳表证、太阳中风、少阳半表半里、阳明经证、阳明腑证
- 架构支持扩展更多阶段

### 6. 历史感冒
- 列表展示所有历史记录
- 点击进入查看详情
- 支持导出、导入 JSON

### 7. 数据导出
- **JSON 导出**：完整备份，供导入恢复使用
- **文字导出**：纯客观记录，给医生看，不包含程序推断

### 8. 自动结束
- 感冒创建后 14 天自动标记为"已结束"
- 支持手动提前结束

## 页面与路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 首页 | 自动判断：有感冒→详情；无感冒→历史列表 |
| `/add-symptom` | 添加症状 | 多选 + 体温 + 时间 + 备注 |
| `/add-medication` | 添加服药 | 选药 + 剂量 + 时间 |
| `/add-note` | 添加备注 | 纯文本备注 |
| `/export` | 导出数据 | JSON / 文字 导出 + JSON 导入 |

## 药品状态逻辑

基于最近一次服药时间，实时计算：

| 状态 | 判定条件 | 显示 |
|------|---------|------|
| 待生效 | 当前 < 起效时间 | "还有 X 分钟生效" + 预计生效时间 X:X |
| 生效中 | 起效时间 ≤ 当前 < 失效时间 | "生效中，还剩 X 小时 X 分" |
| 间隔期 | 失效时间 ≤ 当前 < 下次可服 | "距离下次可服还有 X 小时" + 具体时间 X:X |
| 可服用 | 当前 ≥ 下次可服 | "可服用" |

## 伤寒论推断规则（基础版）

采用**配置驱动 + 策略模式**，支持后续扩展。

### 推断引擎逻辑

```
取最新症状记录
  ↓
按优先级顺序匹配阶段规则
  ↓
返回匹配的阶段 + 代表方
```

### 基础阶段规则

| 阶段 | 核心条件 | 代表方 |
|------|---------|--------|
| **太阳表证** | 恶寒 + 发热（无汗）| 麻黄汤 |
| **太阳中风** | 发热 + 汗出 + 恶风 | 桂枝汤 |
| **少阳半表半里** | 往来寒热（寒一阵热一阵交替）| 小柴胡汤 |
| **阳明经证** | 大热 + 大汗 + 大渴 | 白虎汤 |
| **阳明腑证** | 便秘 + 腹痛 + 发热 | 承气汤 |

### 退变识别
- 阳明证后又出现恶寒 → 提示"坏病"（表里同病）

### 扩展性
- 新增阶段只需添加配置，不改核心代码
- 每个阶段包含：名称、匹配条件、优先级、代表方

## 数据模型

### 感冒记录（Cold）

```typescript
interface Cold {
  id: string;                // UUID
  start_time: string;        // ISO8601
  end_time: string | null;   // ISO8601，null 表示进行中
  status: 'active' | 'completed';
  entries: Entry[];          // 症状/服药/体温/备注记录
}
```

### 条目（Entry）

```typescript
interface Entry {
  id: string;
  timestamp: string;           // ISO8601
  type: 'symptom' | 'medication' | 'temperature' | 'note';
  // type = 'symptom'
  symptoms?: string[];         // 症状列表
  // type = 'medication'
  drug?: string;              // 药品名称
  dose?: string;               // 剂量
  // type = 'temperature'
  value?: number;              // 体温值
  unit?: 'celsius';            // 单位，默认摄氏度
  // type = 'note'
  content?: string;             // 备注内容
  notes?: string;               // 通用备注
}
```

### 应用状态（localStorage）

```typescript
interface AppState {
  version: string;            // 数据版本，用于迁移
  colds: Cold[];               // 所有感冒记录
  settings: {
    auto_end_days: number;     // 默认 14
    default_drugs: Drug[];     // 内置药品配置
  };
}
```

## 导出格式

### JSON 导出（备份恢复用）

```json
{
  "schema_version": "1.0",
  "export_type": "medication-tracker-backup",
  "export_time": "2026-07-08T10:00:00Z",
  "records": [
    {
      "cold_id": "uuid-123",
      "start_time": "2026-07-08T09:00:00",
      "end_time": "2026-07-22T09:00:00",
      "status": "completed",
      "entries": [...]
    }
  ]
}
```

**导入行为：**
- 校验 `schema_version` 和 `export_type`
- 解析后合并到现有数据
- 导入后根据症状时间轴重新推断阶段

### 文字导出（医生版，纯记录）

```
=== 感冒记录 ===
患者自填，供参考

【基本信息】
开始时间：2026-07-08 09:00
记录结束：2026-07-22 09:00（自动结束）

【症状记录】
07-08 09:00 - 恶寒、头痛
07-08 11:00 - 发热（体温38.5℃）
07-08 16:00 - 发热（体温39.0℃）

【体温记录】
07-08 11:00 - 38.5℃
07-08 16:00 - 39.0℃

【服药记录】
07-08 10:30 - 对乙酰氨基酚 0.5g

【备注】
07-08 晚上睡不好，发热反复。
```

> 纯客观记录，不包含任何程序推断。

## 技术栈

| 层级 | 选择 | 原因 |
|------|------|------|
| 框架 | **Vue 3** + **Vite** | 中文社区最活跃，AI 训练数据极丰富 |
| 样式 | **Tailwind CSS** | 原子化 CSS，AI 支持极好 |
| 数据存储 | **localStorage**（JSON 序列化）| 简单直接，AI 几乎不出错 |
| 时间处理 | **Day.js** | 轻量，AI 熟悉度高 |
| 状态管理 | **Vue Composition API** | 原生支持，无需额外库 |
| 图标 | **Lucide Vue** | 轻量，流行 |
| 路由 | **Vue Router** | Vue 生态标准 |

## 交互设计

| 场景 | 行为 |
|------|------|
| 打开首页 | 自动判断：有感冒→详情；无感冒→历史列表 |
| 底部悬浮按钮 | 快速操作：添加症状 / 添加服药 / 添加备注 / 导出 |
| 时间选择 | 默认当前时间，可选自定义时间 |
| 药品选择 | 下拉选择内置药品，自动带出剂量建议 |
| 自动结束 | 创建后 14 天自动标记完成，可手动提前结束 |
| 导出数据 | 底部悬浮按钮触发，选择 JSON / 文字 |
| 导入数据 | 从导出页面入口，上传 JSON 文件 |
| 删除记录 | 左滑或长按条目触发删除确认 |

## 异常处理

| 场景 | 处理 |
|------|------|
| localStorage 满 | 提示"存储空间不足，请导出旧数据" |
| 导入 JSON 格式错误 | 提示"文件格式不正确，请检查" |
| 导入数据版本不匹配 | 尝试自动迁移，失败则提示 |
| 时间选择在未来 | 禁止选择，提示"不能选择未来时间" |
| 同一药品未到间隔期 | 服药时提示"距上次服用不足 X 小时，确认继续？" |

## 扩展性设计

### 阶段规则扩展
- 阶段规则以配置数组形式存储
- 新增阶段：添加配置项即可
- 无需修改核心推断代码

### 药品种类扩展
- 药品配置存储在 app state 中
- 未来可支持用户自定义添加新药品

### 数据迁移
- 每次读取时检查 `version` 字段
- 版本不匹配时执行迁移脚本

## 非功能需求

- **响应式设计**：适配 320px ~ 480px 手机屏幕
- **性能**：localStorage 读写 < 10ms，页面加载 < 1s
- **离线可用**：纯前端，无网络也能正常使用
- **数据安全**：数据仅存储在本地浏览器，不上传服务器
- **可访问性**：触摸目标 ≥ 44x44px，字体 ≥ 16px
